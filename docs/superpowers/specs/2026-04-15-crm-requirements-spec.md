# CRM 系统需求规范

## 1. 项目概述

### 1.1 项目目标

构建一个面向企业销售与客户运营场景的 CRM（客户关系管理）系统，用于帮助企业管理客户信息、跟踪销售过程，并通过数据分析提升销售效率与管理决策能力。

### 1.2 核心应用场景

- 客户信息统一管理
- 销售机会与跟进流程管理
- 数据统计分析与业务决策支持

---

## 2. 用户角色

| 角色 | 描述 |
|------|------|
| 销售人员 | 系统唯一用户角色。负责录入客户信息、创建和推进销售机会、记录跟进内容、查看数据看板。MVP 为单用户系统，无需认证和权限管理。 |

---

## 3. 用户故事与验收标准

### 3.1 客户管理

#### US-1.1 创建客户

> 作为销售人员，我希望录入新客户的基本信息，以便统一管理客户资源。

**验收标准：**

- **AC-1.1.1** Given 我在客户列表页，When 我点击"新建客户"并填写名称、行业、规模、来源（必填），以及地区、地址、备注（选填），Then 系统创建客户记录，状态默认为"潜在"，并跳转到客户列表。
- **AC-1.1.2** Given 我提交表单时必填字段为空，Then 表单显示校验错误提示，不允许提交。

**字段说明：**

| 字段 | 类型 | 必填 | 约束 | 默认值 |
|------|------|------|------|--------|
| 名称 | 字符串 | 是 | 最长 100 字符 | — |
| 行业 | 字符串 | 是 | 最长 50 字符 | — |
| 规模 | 字符串 | 是 | 最长 20 字符 | — |
| 来源 | 字符串 | 是 | 最长 50 字符 | — |
| 状态 | 字符串 | 否 | 最长 20 字符 | 潜在 |
| 地区 | 字符串 | 否 | 最长 100 字符 | — |
| 地址 | 字符串 | 否 | 最长 200 字符 | — |
| 备注 | 文本 | 否 | 无限制 | — |

---

#### US-1.2 查看客户列表

> 作为销售人员，我希望浏览所有客户并按条件筛选，以便快速找到目标客户。

**验收标准：**

- **AC-1.2.1** Given 系统中有客户数据，When 我进入客户列表页，Then 看到按创建时间倒序排列的客户列表，支持分页（每页 20 条）。
- **AC-1.2.2** Given 我输入关键词，When 执行搜索，Then 列表仅显示名称包含该关键词的客户。
- **AC-1.2.3** Given 我选择行业筛选条件，When 执行筛选，Then 列表仅显示该行业的客户。
- **AC-1.2.4** Given 我选择状态筛选条件，When 执行筛选，Then 列表仅显示该状态的客户。
- **AC-1.2.5** Given 筛选条件组合使用，Then 结果为各条件的交集。

---

#### US-1.3 查看客户详情

> 作为销售人员，我希望查看某个客户的完整信息，包括其联系人和关联的销售机会。

**验收标准：**

- **AC-1.3.1** Given 我在客户列表中点击某客户，Then 进入详情页，展示客户基本信息、联系人列表和关联的销售机会列表。

---

#### US-1.4 编辑客户

> 作为销售人员，我希望修改客户信息，以便保持数据准确。

**验收标准：**

- **AC-1.4.1** Given 我在客户详情页点击"编辑"，When 修改字段并提交，Then 客户信息更新成功，页面刷新显示最新数据。
- **AC-1.4.2** Given 我仅修改部分字段，Then 未修改的字段保持原值不变（部分更新）。

---

#### US-1.5 删除客户

> 作为销售人员，我希望删除不再需要的客户记录。

**验收标准：**

- **AC-1.5.1** Given 我在客户详情页点击"删除"，Then 系统弹出确认提示。
- **AC-1.5.2** Given 我确认删除，Then 系统删除该客户及其关联的所有联系人和销售机会（含跟进记录），执行级联删除。

---

#### US-1.6 管理联系人

> 作为销售人员，我希望为客户添加、编辑和删除联系人，以便记录关键对接人信息。

**验收标准：**

- **AC-1.6.1** Given 我在客户详情页，When 我点击"添加联系人"并填写姓名（必填）、职位、电话、邮箱、是否主要联系人（选填），Then 联系人创建成功并关联到该客户。
- **AC-1.6.2** Given 我编辑已有联系人信息并提交，Then 变更即时生效。
- **AC-1.6.3** Given 我删除某联系人，Then 该联系人记录从系统中移除。

**联系人字段说明：**

| 字段 | 类型 | 必填 | 约束 |
|------|------|------|------|
| 姓名 | 字符串 | 是 | 最长 50 字符 |
| 职位 | 字符串 | 否 | 最长 50 字符 |
| 电话 | 字符串 | 否 | 最长 20 字符 |
| 邮箱 | 字符串 | 否 | 最长 100 字符 |
| 主要联系人 | 布尔 | 否 | 默认 false |

---

### 3.2 销售机会管理

#### US-2.1 创建销售机会

> 作为销售人员，我希望为客户创建销售机会，以便跟踪潜在交易。

**验收标准：**

- **AC-2.1.1** Given 我在销售机会列表页或客户详情页，When 我点击"新建机会"并填写标题、关联客户（必填），以及金额、预计成交日期、优先级、备注（选填），Then 系统创建机会记录，阶段默认为"初步接触"，优先级默认为"中"。
- **AC-2.1.2** Given 我提交表单时标题或关联客户为空，Then 表单显示校验错误提示，不允许提交。

**字段说明：**

| 字段 | 类型 | 必填 | 约束 | 默认值 |
|------|------|------|------|--------|
| 标题 | 字符串 | 是 | 最长 200 字符 | — |
| 关联客户 | 外键 | 是 | 必须为已存在的客户 ID | — |
| 阶段 | 枚举 | 否 | 见阶段枚举 | 初步接触 |
| 金额 | 浮点数 | 否 | ≥ 0 | 0 |
| 预计成交日期 | 日期 | 否 | — | — |
| 优先级 | 枚举 | 否 | 高 / 中 / 低 | 中 |
| 备注 | 文本 | 否 | 无限制 | — |

**阶段枚举值（按流程顺序）：**

```
初步接触 → 需求确认 → 方案报价 → 商务谈判 → 赢单 / 输单
```

---

#### US-2.2 查看销售机会列表

> 作为销售人员，我希望浏览所有销售机会并按条件筛选，以便掌握销售全局。

**验收标准：**

- **AC-2.2.1** Given 系统中有机会数据，When 我进入机会列表页，Then 看到按创建时间倒序排列的机会列表，每条记录显示关联客户名称，支持分页。
- **AC-2.2.2** Given 我按阶段筛选，Then 列表仅显示该阶段的机会。
- **AC-2.2.3** Given 我按优先级筛选，Then 列表仅显示该优先级的机会。

---

#### US-2.3 查看销售机会详情

> 作为销售人员，我希望查看某个销售机会的完整信息和跟进历史。

**验收标准：**

- **AC-2.3.1** Given 我在机会列表中点击某个机会，Then 进入详情页，展示机会基本信息和按时间倒序排列的跟进记录列表。

---

#### US-2.4 编辑销售机会

> 作为销售人员，我希望更新机会信息（包括推进阶段），以便反映最新进展。

**验收标准：**

- **AC-2.4.1** Given 我在机会详情页编辑信息，When 修改阶段、金额、优先级等字段并提交，Then 信息更新成功。
- **AC-2.4.2** Given 我仅修改部分字段，Then 未修改的字段保持原值不变（部分更新）。

---

#### US-2.5 删除销售机会

> 作为销售人员，我希望删除无效的销售机会。

**验收标准：**

- **AC-2.5.1** Given 我在机会详情页点击"删除"，Then 系统弹出确认提示。
- **AC-2.5.2** Given 我确认删除，Then 系统删除该机会及其所有跟进记录（级联删除）。

---

#### US-2.6 记录跟进

> 作为销售人员，我希望为销售机会添加跟进记录，以便留存沟通历史。

**验收标准：**

- **AC-2.6.1** Given 我在机会详情页，When 我点击"添加跟进"并填写跟进方式（必填）、跟进内容（必填），以及下一步计划（选填），Then 记录创建成功并显示在跟进历史中。
- **AC-2.6.2** Given 我提交时跟进方式或内容为空，Then 表单显示校验错误提示，不允许提交。

**跟进记录字段说明：**

| 字段 | 类型 | 必填 | 约束 |
|------|------|------|------|
| 跟进方式 | 枚举 | 是 | 电话 / 邮件 / 拜访 / 会议 |
| 跟进内容 | 文本 | 是 | 无限制 |
| 下一步计划 | 文本 | 否 | 无限制 |

### 3.3 数据看板

#### US-3.1 查看业务概览

> 作为销售人员，我希望在首页看到关键业务指标，以便快速了解整体状况。

**验收标准：**

- **AC-3.1.1** Given 我进入数据看板页，Then 看到四个统计卡片：
  - 客户总数：系统中所有客户的数量
  - 活跃机会数：阶段不为"赢单"或"输单"的机会数量
  - 本月新增客户数：当月创建的客户数量
  - 销售总金额：所有机会的金额总和

---

#### US-3.2 查看客户行业分布

> 作为销售人员，我希望了解客户的行业分布情况，以便识别优势行业。

**验收标准：**

- **AC-3.2.1** Given 看板页加载完成，Then 展示客户行业分布饼图，按行业分组统计客户数量。

---

#### US-3.3 查看销售漏斗

> 作为销售人员，我希望看到各阶段的机会分布，以便发现销售瓶颈。

**验收标准：**

- **AC-3.3.1** Given 看板页加载完成，Then 展示销售漏斗图，按阶段顺序（初步接触 → 需求确认 → 方案报价 → 商务谈判 → 赢单 → 输单）显示每个阶段的机会数量和金额。
- **AC-3.3.2** Given 某阶段无机会数据，Then 该阶段显示数量为 0、金额为 0。

---

#### US-3.4 查看金额趋势

> 作为销售人员，我希望看到近 6 个月的销售金额变化趋势。

**验收标准：**

- **AC-3.4.1** Given 看板页加载完成，Then 展示近 6 个月的销售金额折线图，按月汇总机会创建时的金额。

---

#### US-3.5 查看客户增长趋势

> 作为销售人员，我希望看到近 6 个月的客户增长情况。

**验收标准：**

- **AC-3.5.1** Given 看板页加载完成，Then 展示近 6 个月的新增客户柱状图，按月统计新创建的客户数量。

## 4. 边界情况与异常处理

| # | 场景 | 预期行为 |
|---|------|----------|
| E-1 | 删除有关联机会和联系人的客户 | 级联删除所有关联联系人、销售机会及其跟进记录 |
| E-2 | 删除有跟进记录的销售机会 | 级联删除所有关联跟进记录 |
| E-3 | 客户列表/机会列表为空 | 显示空状态提示，引导用户创建第一条记录 |
| E-4 | 分页页码超出实际范围 | 返回空列表，`total` 字段为实际总数 |
| E-5 | 金额输入负数或非数字 | 前端表单校验拦截，不允许提交 |
| E-6 | 客户名称重复 | 允许创建（MVP 不做唯一性约束） |
| E-7 | 预计成交日期早于今天 | 允许设置（兼容历史数据录入场景） |
| E-8 | 近 6 个月内某月无数据 | 图表中该月数据点显示为 0 或不显示 |
| E-9 | 访问不存在的客户/机会详情 | 返回 404 错误，前端展示"未找到"提示 |
| E-10 | 必填字段提交空值 | 前端表单校验拦截；后端返回 422 校验错误 |

## 5. MVP 范围定义

### 5.1 包含（In Scope）

| 模块 | 功能 |
|------|------|
| 客户管理 | 客户 CRUD、搜索筛选（关键词 + 行业 + 状态）、分页 |
| 联系人管理 | 联系人 CRUD（从属于客户详情页） |
| 销售机会管理 | 机会 CRUD、阶段推进、筛选（阶段 + 优先级）、分页 |
| 跟进记录管理 | 跟进记录创建和查看（从属于机会详情页） |
| 数据看板 | 4 个统计指标卡片 + 4 个图表（饼图、漏斗图、折线图、柱状图） |
| 种子数据 | 预置 20 个客户 + 30 个销售机会的演示数据 |

### 5.2 不包含（Out of Scope）

| 功能 | 排除原因 |
|------|----------|
| 用户认证与权限管理 | MVP 为单用户系统，无需登录 |
| 审批流程 | 增加复杂度，核心业务流程不依赖审批 |
| 消息通知与提醒 | 非核心功能，可后续迭代 |
| 文件上传与附件管理 | 增加存储复杂度 |
| 数据导入与导出（Excel/CSV） | 可后续扩展 |
| 操作日志与审计追踪 | MVP 阶段不需要 |
| 多语言支持 | MVP 仅支持中文 |
| 移动端适配 | MVP 仅面向桌面浏览器 |

## 6. 数据模型约束

### 6.1 实体关系

```
Customer (1) ──── (N) Contact
Customer (1) ──── (N) Opportunity
Opportunity (1) ── (N) FollowUp
```

### 6.2 级联规则

- 删除 Customer → 级联删除其所有 Contact 和 Opportunity（含 FollowUp）
- 删除 Opportunity → 级联删除其所有 FollowUp

### 6.3 时间戳

- Customer 和 Opportunity 自动维护 `created_at` 和 `updated_at`
- FollowUp 自动维护 `created_at`
- Contact 无时间戳字段

### 6.4 业务枚举值

| 枚举 | 可选值 |
|------|--------|
| 客户状态 | 潜在、活跃、成交、流失 |
| 机会阶段 | 初步接触、需求确认、方案报价、商务谈判、赢单、输单 |
| 机会优先级 | 高、中、低 |
| 跟进方式 | 电话、邮件、拜访、会议 |
